Centro CRM

Gestión Interna Automatizada

Descripción del Caso

CRM a medida diseñado para digitalizar y automatizar los procesos operativos de Grupo Docente Centro. Su principal objetivo es erradicar el trabajo manual en el control de matrículas y la entrega de certificaciones.

Medidas de Ciberseguridad (Foco OWASP Top 10)

  • Previene SQLi: Uso estricto de consultas preparadas y el ORM de SQLAlchemy.
  • Protección de Sesión: Configuración de cookies con banderas Secure y HttpOnly.
  • Control de Accesos (RBAC): Roles diferenciados para restringir funciones financieras y de modificación a perfiles no autorizados.
  • Prevención CSRF: Implementación de tokens de validación cruzada en todas las peticiones POST/PUT.

Nota de Privacidad

Al ser un CRM que gestiona información privada y facturación interna de la empresa, el código fuente y las bases de datos están restringidos bajo NDA. La interfaz que ves al lado es una recreación funcional para fines demostrativos.

TuProfe EdTech

Simulador de Exámenes de Nombramiento

Descripción del Proyecto

Sistema de preparación y entrenamiento interactivo para docentes a nivel nacional en las evaluaciones oficiales del MINEDU. Admite un banco de preguntas dinámico y genera puntajes en tiempo real con estadísticas.

Aspectos Técnicos & Rendimiento

  • Frontend Astro (SSG/Hydration): Generación estática que garantiza que el portal cargue en menos de 0.8s, usando componentes hidratados solo para la ejecución del examen.
  • Base de Datos Relacional: Diseñado en MySQL con relaciones complejas para mantener subcategorías de temario y rúbricas de respuesta.
  • Motor de Control de Tiempos: Lógica en el backend de Flask para sincronizar el tiempo del usuario, evitando reinicios de examen en caso de desconexiones temporales.

SeparaYA

Gestión de Campos Deportivos

Descripción del Proyecto

Plataforma de consumo público que permite la verificación de disponibilidad por bloques de horarios y la reserva digital instantánea de lozas y campos deportivos.

Solución a Retos Complejos

  • Evita Reservas Duplicadas: Implementación de cierres de transacciones (concurrencia) en base de datos. Cuando dos personas pulsan "Reservar" al mismo milisegundo, el servidor encola la petición y confirma únicamente al primer pago verificado.
  • Diseño Mobile First: Estructura maquetada de manera altamente responsiva para permitir reservas rápidas desde dispositivos móviles a minutos de iniciar un encuentro deportivo.
  • API Restful: Diseño de endpoints modulares en Flask que procesan horarios, estados de canchas, y perfiles de complejos deportivos.
